# {{TITLE}} — desktop app

This directory is a self-contained Electron project generated by Lamp's
Lighthouse. The game lives in `app/` (the same bundle a web build ships);
`main.js` is the Electron main process and `preload.js` the save bridge.

## Run it

```
npm install
npm start
```

or, without installing anything into this directory:

```
npx electron .
```

SAVE and RESTORE use your OS's file dialogs and write ordinary `.lampsave`
files (Documents by default; the panels remember the last folder you used).

**Self-check** (opens no window): `LAMP_SMOKE=1 npx electron .` prints one
diagnostic line and exits 0 if the game boots with working cross-origin
isolation, SharedArrayBuffer, and saves.

**Gotcha:** from a VSCode-hosted terminal, run `unset ELECTRON_RUN_AS_NODE`
first — VSCode exports it, which makes Electron behave as plain Node and the
app fail at startup.

## Package it

The simplest path to a double-clickable app:

```
npx @electron/packager . --out=release
```

This emits `release/<name>-<platform>-<arch>/` containing the app for the
machine you're on. Useful flags:

- `--platform=darwin|win32|linux` and `--arch=x64|arm64` to target others
  (macOS apps can only be *signed* on macOS)
- `--icon=<file>` — `.icns` on macOS, `.ico` on Windows
- `--asar` — pack the sources into an archive so casual browsing doesn't
  expose them (the game text is still best protected by building with
  `--encode-strings`)
- `--overwrite` — replace a previous build

For real installers (DMG, NSIS) and auto-update, step up to
[electron-builder](https://www.electron.build) — heavier and config-driven,
but the standard tool for that job.

## Sharing builds — the honest part

An unsigned app runs fine on the machine that built it, but not necessarily
elsewhere:

- **macOS:** downloaded copies are quarantined by Gatekeeper — players see
  "is damaged and can't be opened". Real distribution requires an Apple
  Developer account, code signing, and notarization (`@electron/osx-sign` and
  `notarytool`; electron-builder wires both). To test a downloaded build
  locally you can strip quarantine with `xattr -cr "<the .app>"` — do not ask
  players to do this.
- **Windows:** SmartScreen interposes a warning on unsigned downloads; a code
  signing certificate removes it.
- **The pragmatic middle ground:** distribute through [itch.io](https://itch.io)
  — its app installs and launches builds in a way players don't fight, without
  you owning certificates.
